À propos de cette page

Cette page permet d'évaluer le comportement du modèle sur un échantillon de données issues de l'ensemble train ou test.

Important : le modèle n'est pas réentraîné lorsque vous cliquez sur "Recalculer". Seul l’échantillon de données utilisé pour l’évaluation change.
Ce que vous pouvez faire
  • Choisir le jeu de données :
    • train : données utilisées pour entraîner le modèle
    • test : données jamais vues pendant l'entraînement
  • Limiter le nombre de points : permet de tracer les graphiques plus rapidement et de garder une bonne lisibilité.
  • Changer la seed : modifie l’échantillon aléatoire sélectionné dans le jeu de données. Le modèle reste identique.
  • Changer le nombre de colonnes : ajuste uniquement la disposition des graphiques.
Interprétation des résultats
  • Predicted vs True : chaque point représente une prédiction du modèle sur une observation réelle. Plus les points sont proches de la diagonale rouge, plus la prédiction est précise.
  • MAE : erreur moyenne absolue, exprimée dans les unités de la variable.
  • RMSE : erreur quadratique moyenne, plus sensible aux grosses erreurs.
  • R² : proportion de variance expliquée par le modèle.
    • 1 = prédiction parfaite
    • 0 = modèle équivalent à la moyenne
    • < 0 = modèle mauvais
  • Histogramme des résidus : permet de visualiser la distribution des erreurs. Une distribution centrée autour de 0 indique un modèle bien calibré.
But de cette page
  • visualiser rapidement les performances du modèle
  • comparer train vs test
  • détecter un surapprentissage (overfitting)
  • comprendre le comportement du modèle sur des données réelles
